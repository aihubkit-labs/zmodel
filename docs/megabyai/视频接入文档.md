外部调用文档
Base URL:
https://newapi.megabyai.cc
所有 API 请求默认使用 Bearer Token 鉴权：
Authorization: Bearer YOUR_API_KEY
请求体与响应体均为 JSON，除视频内容直出接口外。
1. 查询模型
GET /v1/models
示例：
curl https://newapi.megabyai.cc/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
响应示例：
{
  "object": "list",
  "data": [
    {
      "id": "videos-standard",
      "object": "model",
      "owned_by": "video-api",
      "description": "Standard video generation via async /v1/videos"
    },
    {
      "id": "videos-fast",
      "object": "model",
      "owned_by": "video-api",
      "description": "Fast video generation via async /v1/videos"
    },
    {
      "id": "videos-mini",
      "object": "model",
      "owned_by": "video-api",
      "description": "Mini video generation via async /v1/videos"
    }
  ]
}
2. 创建视频任务
POST /v1/videos
Content-Type:
content-type: application/json
请求字段：
字段	类型	必填	说明
model	string	是	videos-standard、videos-fast、videos-mini
prompt	string	是	视频提示词
duration	integer	否	视频时长，4-15 秒，默认 5
ratio	string	否	16:9、9:16、1:1，默认 16:9
resolution	string	否	720p、480p，默认 720p
referenceImages	string[]	否	参考图片 URL，最多 9 个，仅支持公开 http/https
referenceVideos	string[]	否	参考视频 URL，最多 3 个，总时长不超过 15 秒
referenceAudios	string[]	否	参考音频 URL，最多 3 个，总时长不超过 15 秒，建议 MP3/WAV
暂不支持：
{
  "first_image": "...",
  "last_image": "..."
}
如果传入首尾帧字段，会返回不支持错误。
纯文本生成示例
curl https://newapi.megabyai.cc/v1/videos \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "videos-mini",
    "prompt": "A cinematic shot of a person walking through a neon city street at night, realistic lighting, handheld camera",
    "duration": 5,
    "ratio": "16:9",
    "resolution": "720p"
  }'
图片 + 音频参考示例
curl https://newapi.megabyai.cc/v1/videos \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "videos-mini",
    "prompt": "Cinematic action scene, realistic motion, dramatic lighting, no blood, no gore",
    "duration": 5,

    "resolution": "720p",
    "referenceImages": [
      "https://example.com/person.jpg"
    ],
    "referenceAudios": [
      "https://example.com/voice.mp3"
    ]
  }'

3. 查询任务状态
GET /v1/videos/{task_id}
示例：
curl https://newapi.megabyai.cc/v1/videos/videos-mini_xxx \
  -H "Authorization: Bearer YOUR_API_KEY"
任务状态：
状态	说明
queued	已入队，等待处理
in_progress	正在处理或上游生成中
completed	已完成
failed	失败
生成中响应示例：
{
  "id": "videos-mini_xxx",
  "task_id": "videos-mini_xxx",
  "object": "video",
  "model": "videos-mini",
  "status": "in_progress",
  "progress": 50,
  "created_at": 1782690295,
  "completed_at": null,
  "seconds": "5",
  "url": null,
  "video_url": null,
  "metadata": {
    "cached": false,
    "cost_credits": 70
  },
  "error": null
}
完成响应示例：
{

  "model": "videos-mini",
  "status": "completed",

建议轮询间隔：
5-10 秒
4. 下载视频内容
GET /v1/videos/{task_id}/content
示例：
curl -L https://newapi.megabyai.cc/v1/videos/videos-mini_xxx/content \
  -o output.mp4
说明：
* 任务必须是 completed
* 返回 video/mp4
* 该接口用于直接播放或下载生成结果
5. 失败响应
任务失败时，查询任务会返回：
{
  "id": "videos-mini_xxx",
  "task_id": "videos-mini_xxx",
  "object": "video",
  "model": "videos-mini",
  "status": "failed",
  "progress": 100,
  "created_at": 1782690000,
  "completed_at": 1782690010,
  "seconds": "5",
  "url": null,
  "video_url": null,
  "metadata": {},
  "error": {
    "message": "素材格式不被支持，请更换素材或转码后重试。",
    "code": "unsupported_material"
  }
}
常见错误码：
code	message 说明
unsupported_material	素材格式不支持
material_policy_violation	图片或素材未通过审核
unsupported_request	请求参数、模型、素材组合不被支持
content_policy_violation	提示词或内容未通过审核
material_limit_exceeded	素材大小或时长超限
upload_failed	素材上传失败
download_failed	素材或结果下载失败
generation_failed	视频生成失败
generation_timeout	视频生成超时
upstream_network_error	上游网络异常
server_error	服务内部异常
6. 素材要求
图片：
* 必须是公网可访问 http/https URL
* 最多 9 张
* 单文件建议不超过 20MB
* 建议使用 JPG/PNG/WEBP
* 避免带防盗链、登录态、短期签名过快过期的链接
视频：
* 必须是公网可访问 http/https URL
* 最多 3 个
* 总时长不超过 15 秒
* 总大小建议不超过 200MB
音频：
* 必须是公网可访问 http/https URL
* 最多 3 个
* 总时长不超过 15 秒
* 总大小建议不超过 50MB
* 推荐 MP3/WAV
* 不建议 OGG，部分上游组合会返回 invalid media type
7. 完整调用流程
1. 调用 POST /v1/videos 创建任务
2. 保存返回的 task_id
3. 每 5-10 秒调用 GET /v1/videos/{task_id} 查询状态
4. 当 status = completed 时，读取 video_url 或 metadata.content_url
5. 调用 /v1/videos/{task_id}/content 下载 MP4
6. 当 status = failed 时，读取 error.code 和 error.message 做业务处理
8. 最小接入示例
API_KEY="YOUR_API_KEY"
BASE_URL="https://newapi.megabyai.cc"

TASK_ID=$(curl -s "$BASE_URL/v1/videos" \
  -H "Authorization: Bearer $API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "model": "videos-mini",
    "prompt": "A cinematic close-up shot, realistic lighting",
    "duration": 5,
    "ratio": "16:9",
    "resolution": "720p"
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['task_id'])")

echo "task_id=$TASK_ID"

while true; do
  RESP=$(curl -s "$BASE_URL/v1/videos/$TASK_ID" \
    -H "Authorization: Bearer $API_KEY")

  echo "$RESP"

  STATUS=$(echo "$RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['status'])")

  if [ "$STATUS" = "completed" ] || [ "$STATUS" = "failed" ]; then
    break
  fi

  sleep 8
done
